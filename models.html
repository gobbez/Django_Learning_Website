Capire i Models in Django

I Models definiti in Django sono il singolo, definitivo fonte di verit√† sulla tua informazione. Contengono i campi essenziali e comportamenti dei dati che stai memorizzando.

L'ORM di Django fornisce un modo elegante per creare una "migrazione di database" senza scrivere SQL.

Definire un Model

Nel file models.py della tua app, puoi definire i tuoi modelli:

from django.db import models

class Post(models.Model):
    title = models.CharField(max_length=200)
    content = models.TextField()
    published_date = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return self.title

Questo model rappresenta un post del blog con titolo, contenuto e data di pubblicazione.

Migrazione del Database

Dopo aver creato il model, esegui:

python manage.py makemigrations
python manage.py migrate

Questo crea e applica le modifiche al database.
